7 个不同工作场景该用哪些 Skill？一文看懂

AI教程1个月前发布 AI小集
0 3

新一期干货来了~

分享一篇普通人在不同工作场景下，应该使用哪些 Skill 或者 Skill 组合。

Skill 把重复步骤固定下来。一套已经写好的工作方法，什么时候触发、按什么顺序做、输出什么格式。

所以这篇文章可以直接找到自己的场景，照着安装其中的 Skill ，用提示词跑一遍完整任务。

可以在任何支持 Skill 能力的 Agent 上使用、执行任务

我就以使用 Codex 为例。

基础通用

第一个要安装就是 find-skills ，可以自动帮你查找所需要的 skill，比如任务需要专门的能力，就可以先用 find-skills 来搜索是否有更合适的专用 Skill。

另一个安装 skill-creator ，有一天你想把自己的重复工作流写成 Skill，就可以用这个技能，是 Claude 官方推出的，用口语化描述就可以。

办公效率

适合每天要处理文档、表格、会议纪要和网页资料等的人，比如行政、运营同学。入口大多数是文档或表格，出口通常是周报、任务表、汇报材料等。就适合调用这组 Skills：pdf、docx、xlsx、pptx、browser-use/agent-browser、memory。

参考提示词和 SKills：

小小提醒一下：使用 Skill 前要先安装上，安装也很简单，直接给 Agent 对话说，例如，“帮我安装‘技能名称’ skill”。

请按我的办公助理工作流处理这些资料。

1. 用 pdf/docx 分别读取并摘要每份文档，不要混在一起。

2. 用 browser-use/agent-browser 打开并核对相关网页或后台信息，补充文档里没有的上下文。

3. 用 xlsx 把任务、风险、负责人和截止时间整理成表格，字段为：事项、负责人、截止时间、依赖、风险等级。

4. 用 pptx 把关键结论整理成适合汇报的页面大纲。

5. 生成一版适合发给主管的周报，语气简洁，不要写成宣传稿。

6. 用 memory 记住我的周报格式、汇报口径和常用字段。

如果资料里没有明确负责人或截止时间，请标注“待确认”。

安装后再输入提示词，另外每个 Skill 都是可以单独使用。

推荐刚开始使用的时候，分步操作，先用 1-2 两个 Skill ，验证没有问题后，再组合使用。

下面同理，也可以根据自己的工作流和需求删减。

内容创作

自媒体博主、长期写公众号、小红书等内容创作的人。内容创作不单是写一篇文章就完事了，真正考验和麻烦的是选题、搜索、正文、排版、分发等，下面这组 Skills 覆盖了整个工作流。

比如我要做一篇 Claude Code 如何接入飞书？的内容创作。

参考提示词和 Skills：

我要围绕这个主题做一篇内容：[主题]。请按内容创作工作流执行。

1. 用 content-strategy 判断目标读者、内容目标、内容支柱和最值得写的角度。

2. 用 seo-audit 判断搜索意图、标题结构、关键词覆盖和读者最想解决的问题。

3. 用 ai-seo 补充适合 AI 搜索引用的实体、问答、摘要和事实表达。

4. 用 humanizer-zh 把文章去掉AI 味 + 润色 + 去掉废话。

5. 用 baoyu-format-markdown 把正文文章或 Markdown 变成结构清晰、适合阅读的 Markdown。

6. 用 copywriting 写 10 个标题，分别偏 SEO、情绪共鸣、实用清单、争议观点，并写开头和 CTA。

7. 用 social-content 把正文改写成 3 个可分发版本：公众号摘要、小红书笔记、LinkedIn 帖子。

使用 Codex 时，把计划模式打开，生成内容创作计划，人工审核没有问题或者修改后，再开始执行。

Vibe Coding

想用 AI 把自己的想法做成 Demo 的人，推荐用这组里的 Skills。

用 brainstorming 把想法收敛成一个最小 Demo。

用 frontend-design / ui-ux-pro-max 负责生成页面和交互，不会是千篇一律的 AI 蓝紫。

用 web-design-guidelines 检查视觉和布局。

用 test-driven-development 定义最关键的行为。

用 webapp-testing 在浏览器里验证。

用 verification-before-completion 做最后自查。

比如我的想法：电动自行车版顺风车。

产品功能、UI设计、布局都很合理，该重要提醒的也会高亮显示，就是最近电动车戴头盔和带人查的很严，所以算是商业概念原型了。

参考提示词和 Skills

我想用 Vibe Coding 快速做一个 Demo：[一句话描述想法]。

1. 用 brainstorming 先把想法收敛成一个最小可运行版本，只保留最关键的用户流程。

2. 用 frontend-design / ui-ux-pro-max 设计第一版界面、组件和交互，不要做营销页，直接做可用产品界面。

3. 用 web-design-guidelines 检查布局、视觉层级、移动端适配和按钮状态。

4. 用 test-driven-development 定义 3-5 个最关键的行为，先保证核心流程能跑。

5. 实现后用 webapp-testing 在浏览器里检查页面能打开、能操作、没有明显遮挡或空白。

6. 最后用 verification-before-completion 做完成前自查，列出已验证内容和还没覆盖的风险。

营销类

如果是做营销增长，不能只看文案。要先知道用户为什么不转化，再决定改什么，怎么改。

这组 Skills 覆盖的是增长到转化，页面诊断、用户心理、定价、触达、投放。

直接看 Case：

请把自己当成增长顾问，审查这个页面或产品介绍：[https://cakegrowth.com/zh]。 

请按下面流程： 

1. 判断目标用户是谁，当前页面承诺了什么。

2. 用 page-cro 诊断落地页、定价页或注册流程里的转化阻力，找出 10 个转化阻力，按影响程度排序。

3. 用 marketing-psychology 分析用户动机、犹豫点、信任缺口和异议，用营销心理学解释用户为什么会犹豫。

4. 用 pricing-strategy 检查套餐、锚点、试用和优惠设计，给出 A/B 测试假设，包括标题、CTA、价格呈现、信任证明。

5. 用 paid-ads 把核心卖点改写成广告素材，生成一版新版落地页文案。以上用到的技能，自行安装后使用。

6. 最后用 pptx 把内容转成汇报文稿。

招聘/面试

招聘负责人、创业者、团队负责人、面试官，可以为岗位设计专业的面试方案，而且还不能带有偏见，试一下这套工作流 Skill。

请为这个岗位设计面试方案：[岗位描述]。

候选人简历如下：[简历内容]。

请按招聘评估工作流执行：

1. 用 resume-analysis 从简历中提取经历证据、能力线索和需要验证的疑点。

2. 用 interview-designer 提炼岗位成功标准和面试策略。

3. 用 question-bank 设计结构化面试问题，分为过往事实、现场推演、价值观和风险点。

4. 用 scorecard 生成评分表，每项说明高分和低分表现。

面试后

1. 用 meeting-summary 把记录整理成录用建议，但必须区分事实、推断和不确定项。

2. 用 handoff 生成给下一轮面试官的交接摘要。

找工作的小伙伴，完全可以用这种方式，检验一下自己的简历和应当未来会遇到的面试问题。

数据分析

本地数据分析，用 xlsx + pandas-pro 。

xlsx 能读写 .xlsx/.xlsm/.csv/.tsv、本地报表、公式校验，用 pandas 做数据分析，本地 CSV/Excel/JSON 清洗、转换、聚合、大数据优化，更偏工程化数据处理；有向量化、内存优化、验证规范，安全审计通过。

参考提示词：

用 xlsx 读取数据/文件，先做缺失值、异常值和字段含义检查，用 pandas 进行分析，再给结论。

一些心得

Skill 表面上是在安装一个个工具，真正改变的其实是我们处理工作的方式。把反复出现的判断、步骤和标准沉淀下来，让 AI 沿着你已经认可的工作流去完成任务。

真正实际高频使用的 Skill 其实用不了那么多，差不多 10 个左右。不要把技能库装成像百货超市一样。厘清自己的高频场景：哪些事每周都在重复，哪些流程总是容易漏，哪些输出格式希望稳定复用，用 Skill 固定下来，效率才会 up up～

另外 Skill 越强，我们越要检查，要看有没有清楚的 SKILL.md；会不会运行脚本或命令；会不会访问外部网络或第三方 API；会不会访问外部网络或第三方 API。

凡是涉及合同、医疗、财务、权限、删除、付款、发消息、生产环境变更时，让 Agent 先停下来确认，不要一路自动执行。

现在不缺 AI，也不缺大而全的 Skill 清单，不是看谁收藏的工具最多，是谁能把工具变成自己的方法，把方法变成稳定的产出能力。

在真实的工作流里，用真正有用的 Skill 。
